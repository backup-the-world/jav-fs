# 识别规则样例

本文记录 JAV 文件管理中当前已知的文件名识别规则。它面向后续开发者，用来解释测试用例背后的领域含义。

实现和测试主要位于 `src/lib.rs`。

## 视频文件与作品辅助媒体

当前扫描会区分视频文件和作品辅助媒体：

- 视频文件：参与番号 ID 提取、重复扫描、前缀扫描和统计扫描。
- 作品辅助媒体：参与统计扫描，但不参与重复扫描或前缀扫描。

## 番号 ID

番号 ID 用于识别 JAV 作品。

已覆盖样例：

```text
ABC-123.mp4      → ABC-123
XYZ-456.mkv      → XYZ-456
XYZ456.mp4       → XYZ456
ABC789.mkv       → ABC789
ABCD-1234.wmv    → ABCD-1234
AB-12345.mp4     → AB-12345
T38-052.mp4      → T38-052
```

规则要点：

- 支持带横杠和不带横杠的番号 ID。
- 支持大小写混合。
- 纯数字组合不视为番号 ID。
- 文件名中出现多个候选时，优先取更早匹配到的有效番号 ID。

## 下载站前缀

部分文件名会带下载站前缀。当前规则会优先处理 `@` 后面的内容。

```text
hhd800.com@MIDA-307.mp4          → MIDA-307
hhd800.com@ABC123.mp4            → ABC123
4k2.com@13dsvr01794_2_8k.mp4     → dsvr01794
```

没有 `@` 时，仍会在完整文件名中查找番号 ID：

```text
hhd800.com MIDA-307.mp4          → MIDA-307
```

## 后缀和描述文本

番号 ID 后可以跟随额外描述、中文文本或分片样式后缀。提取番号 ID 时只保留作品标识部分。

```text
ABC-123_test.mp4                 → ABC-123
XYZ-123-C.mp4                    → XYZ-123
XYZ-123-中文字符串-C.mkv          → XYZ-123
XYZ-123-中文字符串.mp4            → XYZ-123
XYZ-123-CD1.mp4                  → XYZ-123
```

## 番号前缀

番号前缀是番号 ID 开头的字母部分。

```text
ABC-123   → ABC
mida-983  → mida
star-123  → star
XYZ456    → XYZ
```

如果番号 ID 不以字母开头，则无法提取番号前缀。

## 分片视频

分片视频用于表示同一个 JAV 作品拆成多个视频文件。重复扫描中，不同分片视频不视为重复候选。

已覆盖样例：

```text
4k2.com@13dsvr01794_1_8k.mp4      → 分片 1
twojav.com@urvrsp00535_2_8k.mp4   → 分片 2
XYZ-123-CD1.mp4                   → 分片 1
```

非分片样例：

```text
START-476.mp4
T38-052.mp4
```

## 重复候选判断

重复扫描按番号 ID 聚合同一 JAV 作品的视频文件。

```text
同一番号 ID + 不同分片视频 → 不是重复候选
同一番号 ID + 非分片视频   → 重复候选
```

示例：

```text
4k2.com@13dsvr01794_1_8k.mp4
4k2.com@13dsvr01794_2_8k.mp4
→ 不同分片视频，不是重复候选

START-476.mp4
hhd800.com@START-476.mp4
→ 同一番号 ID，且不像不同分片视频，是重复候选
```

## 未识别视频

如果文件是视频文件，但无法从文件名提取番号 ID，则属于未识别视频。

```text
video.mp4    → 未识别视频
123-456.mp4  → 未识别视频
```
