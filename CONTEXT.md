# JAV 文件管理

JAV 文件管理是围绕 JAV 媒体库中文件识别、扫描、统计与后续整理决策的领域上下文。

## Language

**JAV 文件管理**:
围绕成人影片媒体库中文件识别、扫描、统计与整理决策的领域上下文；项目名称沿用“JAV”，因为目标内容绝大部分是日本成人影片。
_Avoid_: 媒体库扫描、视频文件扫描

**JAV 媒体库**:
被扫描和管理的成人影片文件集合，通常以日本成人影片为主，也可以包含相关图片文件。
_Avoid_: 媒体库、扫描路径、文件集合

**扫描源**:
用户提供给一次扫描的入口位置，可以是本地路径，也可以是 SMB URL。
_Avoid_: 扫描路径、URL_OR_PATH、目标

**JAV 作品**:
由一个番号 ID 标识的成人影片作品；一个 JAV 作品可以对应一个或多个视频文件。
_Avoid_: 视频、影片、番号记录

**番号 ID**:
用于识别一个 JAV 作品的字符串，例如 `MIDA-307`、`T38-052` 或 `DSVR01794`。
_Avoid_: 番号、作品 ID、代码、编码

**番号前缀**:
番号 ID 开头的字母部分，例如 `MIDA-307` 的 `MIDA`；它用于按作品标识的前段进行归类或统计，但不承诺一定代表厂牌或系列。
_Avoid_: 前缀、系列代码、厂牌代码

**作品辅助媒体**:
与 JAV 作品相关但不是作品主体的视频内容的文件，例如封面或截图等图片文件。
_Avoid_: 图片文件、一等媒体文件、扫描噪声

**分片视频**:
同一个 JAV 作品被拆分后形成的多个视频文件之一；不同分片视频属于同一个 JAV 作品，但不应被视为重复。
_Avoid_: 分碟、集、片段、重复变体

**重复候选**:
同一个番号 ID 下出现的多个非分片视频文件；它表示需要进一步确认的潜在重复，而不是已经证明内容完全相同的文件。
_Avoid_: 重复视频、冲突、多余副本

**统计扫描**:
面向 JAV 媒体库规模概览的扫描，关注文件总量、视频文件数量、作品辅助媒体数量以及大小汇总。
_Avoid_: Stats Mode、普通扫描

**重复扫描**:
面向重复候选发现的扫描，按番号 ID 聚合同一 JAV 作品下的视频文件，并排除不同分片视频。
_Avoid_: Duplicate Mode、冲突扫描

**前缀扫描**:
面向番号前缀盘点的扫描，收集 JAV 媒体库中出现过的唯一番号前缀。
_Avoid_: Prefix Mode、系列扫描、厂牌扫描

**未识别视频**:
属于视频文件，但无法从文件名提取番号 ID 的文件。
_Avoid_: 失败视频、未知作品、孤立视频

## Example dialogue

开发者：这个功能属于 JAV 文件管理吗？
领域专家：如果它影响 JAV 媒体库中文件如何被识别、扫描、统计、去重或整理，就属于。

开发者：MIDA-307 是视频文件吗？
领域专家：不是，MIDA-307 标识的是一个 JAV 作品；这个作品可能有一个视频文件，也可能有多个分片视频文件。

开发者：同一个番号 ID 出现两次一定是重复吗？
领域专家：不一定。如果它们是同一个 JAV 作品的不同分片视频，就不是重复；否则才需要按重复候选处理。

开发者：重复候选是不是一定可以删除？
领域专家：不能。重复候选只说明番号 ID 相同且不像不同分片，是否删除还需要确认清晰度、字幕、版本或文件内容。
