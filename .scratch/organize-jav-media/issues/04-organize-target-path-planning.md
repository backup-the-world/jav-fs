# 按数据库元数据生成入库目标路径计划

## Parent

.scratch/organize-jav-media/PRD.md

## What to build

扩展 `organize` dry-run，让它从数据库读取完整 JAV 作品元数据，并生成可执行的入库目标路径计划。计划应包含演员目录、作品目录、目标视频文件名、跳过原因和路径警告，但仍不写文件。

目标路径必须复用正式 JAV 媒体库结构：`#演员名,演员名/[年份] 番号ID - 标题/视频文件`。演员目录名去掉半角/全角空格；多演员优先复用已有同演员集合目录，不存在时按演员 ID 升序生成。作品目录需要清洗非法路径字符，并在过长时截断标题但保留 `[年份] 番号ID - ` 前缀。

## Acceptance criteria

- [x] dry-run 查询视频、演员、发行日期、标题等规划所需元数据。
- [x] 缺少演员信息时跳过并报告“缺少演员信息”。
- [x] 缺少发行日期时跳过并报告“缺少发行日期”。
- [x] 标题为空时跳过并报告“标题为空”。
- [x] 演员目录名前缀为 `#`，演员名去掉半角/全角空格，多演员用逗号连接。
- [x] 已存在同演员集合目录时复用现有目录名。
- [x] 没有现有目录时按演员 ID 升序生成演员目录。
- [x] 作品目录格式为 `[年份] 番号ID - 标题`，番号 ID 统一大写。
- [x] 标题非法路径字符被清洗，并报告路径清洗/截断警告。
- [x] 作品目录过长时截断标题，但保留 `[年份] 番号ID - ` 前缀，并报告路径清洗/截断警告。
- [x] 普通视频目标名为 `番号ID.ext`，分片视频目标名为 `番号ID-CDn.ext`。
- [x] 同一批次内同番号不同分片一起规划入库。
- [x] 同一批次内同番号但无法区分分片的多视频整组跳过并报告批内冲突。
- [x] 目标作品目录已存在时允许规划追加未冲突的新视频。
- [x] 目标同名视频存在时跳过该视频并报告目标同名冲突。
- [x] dry-run 报告展示将入库、批内冲突、目标同名冲突和各类元数据缺失。
- [x] 入库规划模块有单元测试覆盖主要路径和跳过场景。
