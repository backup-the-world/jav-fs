# PRD：JAV 媒体库入库整理

## Problem Statement

用户需要把 incoming 区域中的高清视频文件整理进正式 JAV 媒体库。当前 `jav-fs` 只能执行统计扫描、重复扫描和前缀扫描，不能根据本地数据库元数据完成入库整理，也不能生成 NFO、下载作品辅助媒体或按现有媒体库目录结构移动视频。

用户目前的源目录是 `/Volumes/jav/incoming/incoming-hd/`，目标目录是 `/Volumes/jav/media/hd/`，元数据数据库是 `~/dev/jav/jav-hub/data/jav-data.db`。正式库已有稳定结构：第一层按演员集合命名，第二层按 `[年份] 番号ID - 标题` 命名，作品目录内包含视频、NFO、封面、缩略图、fanart 和 `extrafanart/`。入库整理必须复用这个结构，避免污染正式 JAV 媒体库。

## Solution

新增 `organize` 子命令，用于执行 **入库整理**：递归扫描 incoming 区域中的视频文件，提取番号 ID，查询 SQLite 数据库，基于数据库中的 JAV 作品元数据计算目标路径，生成 NFO，尝试下载作品辅助媒体，并在显式 `--apply` 时把视频移动到正式 JAV 媒体库。

该功能默认 dry-run。dry-run 不写文件、不下载图片、不移动视频、不联网，只输出入库计划、跳过原因和警告。只有传入 `--apply` 才执行实际改动。

整理以 JAV 作品为组织核心，同时保留视频文件和分片视频的独立性。缺少关键元数据、无法识别番号 ID、批内冲突、目标同名冲突等情况应被安全跳过并报告，而不是猜测或覆盖正式库内容。

## Implementation Decisions

- 新增 `organize` 子命令。现有扫描入口保持兼容；整理功能不再通过布尔扫描 flag 表达。
- 默认 dry-run；`--apply` 才执行写文件、下载图片和移动视频。dry-run 不联网、不创建目录、不写 NFO、不下载图片、不移动视频。
- 使用当前工作目录下的本地配置文件提供 `organize` 默认值；配置文件不提交，仓库提供示例配置。
- `source`、`target`、`database` 必须来自一组完整来源：完整 CLI 或完整配置。CLI 完整时优先；部分 CLI 参数报错；不做字段级合并。
- 配置路径支持 `~` 展开，不支持环境变量展开。
- 构建一个较深的入库配置解析模块，输入 CLI 参数和可选配置文件，输出完整、已展开、已校验的入库配置或明确错误。
- 扩展全项目的视频文件识别范围为 `.mp4/.mkv/.wmv/.avi/.mov/.m4v/.ts`，并同步扫描和整理行为。
- 构建一个较深的源扫描模块，负责递归扫描、隐藏文件/目录跳过、内置排除规则和用户追加 exclude 规则，输出候选视频文件列表。
- 入库整理沿用当前番号 ID 提取规则，不处理多候选/合集歧义。提取不到 ID 归为未识别视频；提取到但数据库无记录归为缺少元数据。
- 构建一个数据库元数据模块，负责按番号 ID 查询 JAV 作品、演员、导演、genre、图片 URL 和必要字段完整性。
- 缺少演员信息、发行日期或非空标题时，跳过该 JAV 作品或视频，不进入正式库。
- 构建一个较深的入库规划模块，负责把视频候选和数据库元数据转换为入库计划。该模块应封装演员目录、作品目录、视频 basename、NFO basename、图片目标路径、冲突检测和报告分类。
- 目标演员目录以 `#` 开头；演员名来自数据库但目录名去掉半角/全角空格；多演员逗号连接。优先复用已有同演员集合目录；否则按演员 ID 升序生成。
- 作品目录使用 `[年份] 番号ID - 标题`。路径非法字符清洗；目录组件过长时截断标题但保留前缀；相关情况加入路径清洗/截断警告。
- 目标视频命名规范化：去下载站前缀，番号 ID 大写，普通视频为 `番号ID.ext`，分片为 `番号ID-CDn.ext`，不保留站点名和清晰度噪音。
- 同一批次内按番号 ID 聚合。不同分片一起入库；同番号但无法区分为不同分片的多视频整组跳过并报告批内冲突。
- 已存在作品目录允许追加未冲突的新视频。目标同名视频冲突时跳过该视频，不覆盖视频，也不生成或覆盖该视频对应的 NFO 和图片。
- 构建一个 NFO 生成模块，输入 JAV 作品元数据、视频 basename 和已成功/已存在图片集合，输出现有 XML 风格 NFO 内容。
- 每个视频文件生成同 basename 的 NFO。NFO 标题字段使用 `番号ID - 标题`，不包含年份和分片号。
- NFO 中演员名保留数据库原始写法；目录名去空格仅用于路径。
- 每个数据库 genre 同时写入 `<tag>` 和 `<genre>`。
- NFO 字段映射：`studio = maker 优先，否则 label`，`maker = maker`，`label = label`，`set = series`，`runtime = {duration}分鍾`，`description` 同时写入 outline 和 plot。
- NFO 只引用实际存在或本轮成功下载的图片。NFO 生成失败阻止视频移动。
- 构建一个作品辅助媒体模块，负责按数据库 URL 下载图片、覆盖本次成功入库视频对应的 basename 图片，并报告下载失败。
- 图片字段映射：portrait cover 到 poster；无 portrait 时通用 cover 到 poster；landscape cover 到 fanart；通用 cover 到 thumb；preview images 到作品级 `extrafanart/extrafanart-N.jpg`。
- 图片下载失败不阻止视频入库，只作为图片下载警告。成功入库的边界是视频进入正确作品目录且 NFO 生成成功。
- 追加新分片时，为新分片生成 NFO 和 basename 图片；作品级 `extrafanart/` 已存在则不重复下载，缺失时可尝试下载。
- 执行顺序为：查询和规划、检查冲突、创建目标目录、下载/覆盖图片、生成 NFO、移动视频。图片失败只警告；NFO 失败阻止移动。
- 如果准备阶段失败导致视频不移动，清理本轮新增文件；新建作品目录若最终没有成功入库任何视频，则删除本轮创建的空目录。不恢复已覆盖图片，但必须在报告中体现该边界。
- 构建一个移动执行模块，优先 rename；跨文件系统失败时复制、校验大小一致后删除源文件。删除源文件失败时入库仍算成功，但报告源文件删除失败。
- 报告分类包括：将入库/已入库、已存在跳过、批内冲突、目标同名冲突、缺少元数据、缺少演员信息、缺少发行日期、标题为空、NFO 失败、图片下载警告、路径清洗/截断警告、未识别视频、源文件删除失败。
- 报告先打印到终端。机器可读 JSON 报告是后续增强，不在第一版必须实现。
- 文档需要同步更新 CLI 说明、架构说明和识别规则说明，特别是新增 `organize`、配置文件、dry-run/apply 和视频扩展名变更。

## Testing Decisions

- 测试应覆盖外部行为和稳定接口，不测试内部实现细节。优先测试“给定输入文件名、数据库记录、目标库状态，得到的入库计划/报告/生成内容是否正确”。
- 现有 `src/lib.rs` 中已有文件名识别、视频文件识别、番号 ID 提取、分片识别和 SMB 转换单元测试；扩展视频扩展名和整理命名规则时应沿用这种快速单元测试风格。
- 配置解析模块需要测试：完整配置可用、完整 CLI 优先、部分 CLI 报错、配置不完整报错、`~` 展开、不展开环境变量、exclude 追加。
- 源扫描模块需要测试：隐藏文件/目录跳过、默认排除规则生效、配置和 CLI exclude 追加、只产出支持扩展名的视频文件。
- 视频识别规则需要测试新增扩展名，并确认现有 `.mp4/.mkv/.wmv` 行为不回退。
- 入库规划模块需要重点测试，因为它应承载最多规则且不需要真实移动文件：演员目录生成、复用已有演员集合目录、作品目录生成、标题清洗和截断、视频 basename、分片命名、批内冲突、目标同名冲突、缺少元数据、缺少演员、缺少发行日期、标题为空。
- 数据库元数据模块应使用临时 SQLite 数据库做集成测试，覆盖有完整元数据、缺演员、缺发行日期、缺标题、含 genre、含导演、含图片 URL 的查询结果。
- NFO 生成模块需要纯单元测试，断言 XML 中包含关键字段、CDATA 包裹描述和标题、genre/tag 双写、runtime 风格、图片引用只包含存在/成功图片。
- 图片下载模块应通过可替换 HTTP 客户端或测试替身验证 URL 到目标文件的映射、覆盖行为、失败警告，不依赖真实外网。
- 移动执行模块需要测试 rename 成功、跨卷复制回退、大小校验失败、删除源文件失败报告。跨卷错误可通过抽象文件操作接口注入，避免测试依赖真实挂载卷。
- dry-run 测试应确认不会创建目录、不会写 NFO、不会下载图片、不会移动视频，且报告计划完整。
- apply 测试应使用临时目录和临时数据库，验证成功入库后源视频离开 incoming、目标视频/NFO/图片出现在预期位置。
- 失败清理测试应覆盖 NFO 失败后本轮新增文件清理、新建空作品目录清理，以及不恢复已覆盖图片的报告边界。
- 报告测试应断言每类跳过/警告可被稳定分类，而不是只检查人类可读字符串。
- 回归测试应覆盖现有统计扫描、重复扫描、前缀扫描仍能运行，并使用扩展后的视频文件定义。

## Out of Scope

- 不实现自动补充数据库元数据；数据库缺少记录、演员、发行日期或标题时只报告并跳过。
- 不处理多番号合集或文件名中多个数据库命中的歧义；第一版假设 incoming 文件名只有一个有效番号。
- 不根据文件大小、哈希、清晰度、字幕、破解流出版本等自动判断更优版本。
- 不覆盖目标同名视频。
- 不刷新因目标同名冲突而跳过的视频的 NFO 或图片。
- 不实现交互式确认；dry-run 是确认机制。
- 不在 dry-run 中联网探测图片 URL。
- 不实现机器可读 JSON 报告；仅保留后续扩展空间。
- 不实现媒体库全量重整、重命名已有正式库作品、清理重复候选或删除旧作品辅助媒体。
- 不支持环境变量路径展开。
- 不把用户本机默认路径硬编码进程序。
